# Music Store Database 🎵📀

This repository contains a PostgreSQL database schema for a digital music store. It is designed to handle data related to artists, albums, tracks, customers, and sales transactions.

## 🚀 Project Overview
- Designed a normalized relational database schema with 11 tables.
- Established primary and foreign key constraints to ensure data integrity.
- Populated the database with comprehensive sample data for a realistic querying experience.
- The schema supports operations like browsing music, managing playlists, and processing invoices.

## 📂 Files in this Repository
- `Music_Store_database.sql` → The complete SQL script to create and populate the database.
- `README.md` → Project documentation.

## 🛠️ Tools & Technologies Used
- **Database:** PostgreSQL
- **Language:** SQL

## 🔑 Key Features
- **Structured Schema:** Includes tables for artists, albums, tracks, customers, invoices, and playlists.
- **Data Integrity:** Uses relationships to link tables logically (e.g., tracks to albums, albums to artists).
- **Ready for Analysis:** The included data allows for complex queries to find insights, such as top-selling artists or genres by country.
